How to backup Firestore the Firebase way

Using Cloud Scheduler and Cloud Functions

Humi

Jul 12, 2019·3 min read

Status quo (07/2019)

it is not possible to automate backups of firestore database in firebase console like it is for the realtime database. You have to do it on your own!

And No. No backup is no option :)

Google advises you this to use cronjob + AppEngine + bucket: https://firebase.google.com/docs/firestore/solutions/schedule-export

What's wrong with this idea? Nothing at all but I think we have a better option since we can use Cloud Scheduler (introduced in 2018). We will build a scheduled cloud function that does the job!

Let's start:

Step 1 — Create a Service Account:

  1. Goto https://console.cloud.google.com/iam-admin/serviceaccounts
  2. Click CREATE SERVICE ACCOUNT
  3. Type a name (eg. firestore-backup)and description
  4. Select Role Cloud Datastore Import Export Admin
  5. In the last Step, click CREATE KEY and save it as JSON

Step 2 — Create a new Bucket for Backups

After that, we create have to create a new bucket for our backups

  1. Goto https://console.cloud.google.com/storage/browser
  2. Click CREATE BUCKET
  3. Name: YOUR_PROJECT_ID-firestore-backup
  4. Select Storage Class, Location, ACM and Encryption

Suggestions
Multiregional Class, Same Location as your Firesbase Project,
Object Level ACM, Google Managed Encryption

Step 3 — Add Permission to Service Account

  1. Open your Bucket and go to Permissions.
  2. Click Add Members
  3. Type in the Name of your new service account (eg. firestore-backup@...)
  4. Select the role: Storage Admin

If you like you can add a lifecycle to your bucket to delete backups older then 30 days.

Step 4 — Add Scheduled Cloud Function to run Backup

Now use your service account key JSON and jump to your cloud functions. The example is written in typescript.

In your tsconfig.json add under compilerOptions

"resolveJsonModule": true

Install googleapis :

npm install -s googleapis

What this code does:

  • Create a scheduled job that triggers every day at 12 pm (00:00)
  • That job triggers the cloud function you see here
  • In the function, we call the export documents API method of googleapis
  • This API method does all the work in background and will return within 1 sec

Bonus Tipp:

Big change? Trigger extra backup? Just go to your cloud scheduler tab and click run. Here you go with your extra backup.
